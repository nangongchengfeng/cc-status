# Dashboard Redesign Design

**目标**

将 `web/src/pages/Dashboard/` 重构为一套更适合演示汇报的数据驾驶舱首页。页面需要在首屏快速传达“今天花了多少、趋势怎么走、是谁在消耗、最近发生了什么”，同时保持信息清晰、交互流畅和工程改动可控。

**设计方向**

- 风格：浅蓝指挥舱。
- 气质：冰白、浅蓝、白灰的轻科技风。
- 关键词：不对称、通透、主次分明、适合汇报。
- 记忆点：把页头升级为“主舞台”，让标题、关键数字、范围控制和系统状态在同一视觉区域协同出现。

**适用场景**

该页面主要用于演示汇报，而不是高密度后台管理台。因此需要优先保证：

- 首屏视觉抓力强。
- 核心指标能被非技术角色一眼理解。
- 阅读顺序自然，便于边讲边展开。
- 图表、排行和明细在视觉上有明确层级，不互相抢戏。

**总体策略**

- 不改后端接口和查询 hooks。
- 不引入新的 UI 库，继续使用 `React + Tailwind CSS + Recharts`。
- 主要重构页面编排、卡片样式、全局背景、颜色变量和局部交互反馈。
- 尽量复用现有页面组件的数据输入契约，降低回归风险。

**布局方案**

页面采用不对称的 `12` 列自由网格，形成“左宽右窄”的首屏结构。

- 左上主舞台：展示页面标题、核心说明、时间范围摘要和最重要的概览数据。
- 右上控制塔：展示时间范围切换、当前查询粒度、加载状态和错误状态。
- 中部趋势区：费用趋势为主图，Token 趋势为副图，两块区域宽度不等。
- 底部信息区：左侧纵向堆叠模型排行和客户端排行，右侧展示缓存效益和最近请求。

整体阅读路径固定为：

1. 先看核心成本和总量。
2. 再看趋势变化。
3. 再看谁在消耗。
4. 最后看最近发生了什么。

**视觉系统**

- 背景：浅灰蓝渐变底色，叠加轻微网格纹理和雾状蓝光。
- 卡片：半透明白蓝玻璃面板，搭配柔和阴影和浅色描边。
- 重点区域：使用蓝色发光边框或高亮数字强化视觉重心。
- 次级信息：用冷灰和低对比文字处理，避免与主数据竞争。

页面不使用纯平背景和平均分栏，避免回退为普通后台模板。

**模块调整**

`DashboardPage`

- 负责新的网格编排和首屏信息组织。
- 将错误和加载状态嵌入右上控制塔区域。
- 在页头补充当前时间范围的说明性摘要。

`OverviewCards`

- 从四张平均卡片调整为“主指标 + 次级指标”组合。
- `总费用` 和 `总 Token` 占据更高视觉权重。
- `总请求数` 和 `活跃客户端` 作为次级指标卡承接。

`CostTrendChart` / `TokenTrendChart`

- 统一升级为浅色科技风图表容器。
- 保留现有图表类型和数据结构。
- 调整坐标轴、tooltip、网格线和描边颜色以适配新主题。
- 明确费用趋势为主图，面积和标题级别都更高。

`ModelRanking` / `ClientRanking`

- 保留纵向条形图结构。
- 通过容器尺寸、标题层级和留白提升可读性。
- 条形颜色收敛为蓝青系，减少当前暖色混杂。

`CacheAnalysis`

- 从简单指标盒子升级为辅助分析面板。
- 保持“节省金额”是最先被看到的指标。
- 其余两项作为解释性补充，不喧宾夺主。

`RecentRequestsTable`

- 维持表格语义结构。
- 调整表头、行高、hover 状态和背景层次。
- 将其定位为“底部明细面板”，不与主舞台竞争。

**文案策略**

保留现有短句、口语化表达，但更偏“汇报导向”。

- 标题仍然直接。
- 副标题强调“先看花费，再看来源”。
- 各模块说明句保持简短，长度控制在一到两行。
- 加载和错误提示继续保持轻口语，不做技术化错误描述。

**交互策略**

- 时间范围切换器使用更强的选中态和轻微悬浮反馈。
- 卡片 hover 只做极轻的位移、亮度和边框变化。
- 图表 tooltip 统一为浅蓝玻璃浮层。
- 表格行 hover 强化当前行聚焦感，帮助演示时口述定位。

动画策略偏克制，避免为了“炫”而影响阅读稳定性。

**状态处理**

页面继续覆盖 `loading / error / empty / success` 四种状态：

- `loading`：在控制塔区域提示正在刷新当前范围。
- `error`：在控制塔区域提示数据暂未接上，但保留页面骨架。
- `empty`：各模块继续显示稳定空态文案。
- `success`：显示格式化后的真实数据。

这样可以保持状态稳定性，同时不破坏整体布局。

**实现边界**

- 优先修改 `DashboardPage.tsx` 和页面专属组件，不外溢到无关模块。
- 允许调整 `web/src/styles/index.css` 以重建全局背景和根字体风格。
- 若测试文案因设计重写而变化，需要同步更新页面测试。
- 若图表配色变化导致断言或快照不适配，只调整最小必要测试。

**验证策略**

- 运行 `web/package.json` 中已有的 `npm run test`。
- 运行 `npm run build` 验证页面和样式编译通过。
- 使用诊断工具检查本次编辑文件是否引入 TypeScript 或 JSX 问题。
