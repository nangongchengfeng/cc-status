# Dashboard Static Replica Design

**目标**

在 `web/` 中将首页大屏重构为一版按参考图静态 1:1 高保真复刻的页面。第一阶段只追求视觉结构、色彩、密度、文案位置和细节贴近参考图，不接入真实接口联动，使用本地 mock 数据稳定渲染。

**约束**

- 保持根路由 `/` 不变，仍由 `DashboardPage` 承载首页。
- 第一阶段只做静态高保真复刻，不扩展新业务功能，不修改后端接口。
- 尽量复用现有 React、Tailwind CSS、Recharts、Vitest 工程栈。
- 改动范围控制在 `web/src/pages/Dashboard/`、`web/src/styles/` 和少量格式化辅助，不顺手重构无关模块。
- 页面仍需保留可构建、可测试、可继续回接真实数据的能力。

**总体方案**

首页继续使用 `DashboardPage`，但展示逻辑由“真实查询驱动页面”改为“静态 mock 驱动页面”。页面顶层只保留少量状态与常量，用于支撑截图级复刻所需的时间标签和筛选器外观。现有 `OverviewCards`、趋势图、排行、缓存效益、最近请求等模块会按参考图重新拆分和重写，以获得更接近的空间比例、装饰层级和视觉节奏。

第一阶段不删除既有 dashboard 数据结构与查询代码，只是在页面编排层暂时不依赖它们。这样后续第二阶段重新接回 `useDashboardQuery()` 和 `useRecentLogsQuery()` 时，只需要把 mock 数据替换为真实数据映射，不需要再次推翻页面视觉实现。

**视觉基调**

- 背景切换为深蓝黑渐变，叠加径向冷光、轻噪点和雾化发光层。
- 顶部加入发光球体和紫橙粒子波浪装饰，形成和参考图一致的主视觉重心。
- 卡片容器统一采用半透明深蓝底、冷色描边、模糊叠层和柔和外发光。
- 文字层级按“高亮数字 > 主标题 > 模块标题 > 辅助说明 > 表格弱文案”组织。
- 数值和图表色彩与参考图锁定：主标题偏粉白，费用偏暖黄，缓存效益偏绿青，榜单使用紫色和蓝色两组条形色。

**布局**

- 顶部为横向页头，左侧放标题和副标题，右侧放“数据更新于”状态胶囊。
- 页头中部覆盖装饰层，包括发光球体和横向粒子波浪。
- 主体采用左右不对称双列布局：
- 左列更宽，依次放“核心指标”面板、“使用趋势分析”面板。
- 右列更窄，依次放“综合效益”面板、“最近请求”面板。
- “使用趋势分析”内部再拆成四块：费用趋势、Token 趋势、模型排行、客户端排行。
- 页面底部补一条信息栏，左侧放提示说明，右侧放数据来源与构建信息。

**模块拆分**

- `pages/Dashboard/DashboardPage.tsx`：页面编排、装饰层、静态数据注入。
- `pages/Dashboard/dashboardReplica.mock.ts`：集中管理卡片、图表、榜单、表格所需 mock 数据。
- `pages/Dashboard/components/DashboardHero.tsx`：页头标题、更新时间、顶部装饰。
- `pages/Dashboard/components/MetricsPanel.tsx`：左上核心指标大面板与四个指标卡。
- `pages/Dashboard/components/BenefitPanel.tsx`：右上综合效益面板与三项效益卡。
- `pages/Dashboard/components/TrendPanel.tsx`：费用趋势和 Token 趋势双图区域。
- `pages/Dashboard/components/RankingPanel.tsx`：模型排行和客户端排行双榜单区域。
- `pages/Dashboard/components/RequestsPanel.tsx`：最近请求表格与顶部按钮。
- `pages/Dashboard/components/DashboardFooterMeta.tsx`：底部提示与数据来源条。

**数据策略**

- 页面创建一份结构化 mock 数据，字段命名尽量贴近现有 dashboard 类型，降低后续回接成本。
- 图表数据直接使用静态数组，确保折线峰值、面积堆叠和榜单排序能复现参考图的走势与比例。
- 表格数据使用固定 7-8 行，锁定时间、模型名、输入、输出、费用和客户端字段长度。
- 第一阶段不再根据 query 状态切换 loading/error/empty/success，统一以“成功展示态”为主。

**样式策略**

- 主题背景、噪点、主光晕和全局最小宽度收口到 `src/styles/index.css`。
- 页面专属复杂背景、发光球体、波浪、渐变遮罩等放在 `DashboardPage` 或页面专属组件内，避免污染全局。
- 大量重复的玻璃卡片、标题条、状态胶囊通过小型局部组件复用，不额外抽象成跨页基础组件。
- 精准还原优先于过度抽象，组件拆分以视觉区块为边界。

**测试策略**

- 保留一个最小页面测试，验证根路由仍能渲染标题和关键模块标题。
- 针对静态 mock 页面新增或更新轻量测试，锁住关键文案、更新时间、主要面板标题和最近请求表头。
- 不测试装饰层像素细节，不测试 Recharts 内部实现。
- 继续以 `npm run test` 和 `npm run build` 作为最小回归证据。

**执行方式**

1. 先落地设计文档和实现计划。
2. 先重构 `DashboardPage` 页面骨架与全局背景。
3. 再逐块实现顶部装饰、核心指标、综合效益、趋势、榜单和表格。
4. 最后统一微调间距、发光、边框、字号、表格密度和底部信息条。
5. 静态高保真版本稳定后，再单独规划第二阶段的真实数据回接。
