# Claude Usage Sync Client

用于从 Claude Code 本地会话日志中提取 token 使用数据，并将新记录同步到中心服务的单机场景客户端。

## Language

**会话日志**:
保存在本机 `~/.claude/projects/` 下的 JSONL 文件集合，用于承载 Claude Code 会话消息。
_Avoid_: 聊天文件, 原始日志

**Token 使用记录**:
从单条满足条件的 assistant 消息中提取出的可计量 token 数据单元。
_Avoid_: 统计项, 费用记录

**同步状态**:
客户端为每个会话日志文件维护的增量解析进度，至少包含修改时间和已处理行号。
_Avoid_: 检查点, 游标

**已上报记录**:
客户端确认已成功发送到中心服务、后续不应再次上报的记录集合。
_Avoid_: 缓存命中, 已处理消息

**客户端标识**:
客户端首次生成并持久化的稳定 UUID，用于标识一台安装 Claude Code 的电脑。
_Avoid_: 机器指纹, 临时实例 ID

**应用数据目录**:
客户端统一使用 `~/.cc-usage-client` 维护配置、SQLite 数据库和运行元数据的独立目录。
_Avoid_: 工作目录, Claude 日志目录

**使用记录**:
服务端持久化的单条客户端上报数据，包含请求标识、模型、token 用量、费用和时间戳。
_Avoid_: 代理日志, 请求日志

**上报批次**:
客户端一次 `POST /api/v1/sync` 请求中携带的一组**使用记录**。
_Avoid_: 同步任务, 上传文件
首版要求 `reports` 至少包含 1 条记录；空批次返回 `400`。

**使用记录库**:
服务端用于保存**使用记录**的核心表，统一命名为 `usage_reports`。
_Avoid_: proxy_request_logs, 代理表

**模型定价**:
服务端按模型维护的每百万 token 单价配置，用于计算**使用记录**的费用。
_Avoid_: 默认成本, 静态价格常量
`GET /api/v1/model-pricings` 首版返回全部定价记录，并显式包含 `is_placeholder`，默认定价不做隐藏。
`PUT /api/v1/model-pricings/:id` 首版采用全量更新语义，不做部分更新。
`model_id` 与上报 `model` 首版统一做小写规范化后再存储、匹配和返回。

**默认定价**:
当模型未命中精确或模糊定价时，服务端允许回退到一条全局 placeholder 定价继续计算费用；费用字段统一按 USD 存储。
_Avoid_: 猜测模型价格, 人民币落库
全局 placeholder 默认定价首版强制只能存在一条；创建或更新产生冲突时返回 `409`。

**服务端客户端标识**:
服务端把 `client_id` 视为客户端提供的稳定技术标识，只要求非空和长度受限，不强制 UUID 格式，也不等同于机器名。
_Avoid_: 主机名, 展示名称, 机器别名

**同步兼容响应**:
`POST /api/v1/sync` 为兼容现有 client，成功时返回 `code`、`message`、`accepted_count`、`duplicate_count`，而不是通用的 `{ "data": ... }` 包装。
_Avoid_: 通用成功包装, 200 + 业务失败码
首版对单个请求批次使用单事务；唯一键重复计入 `duplicate_count`，其他数据库错误会回滚整批并返回 `500`。

**趋势桶**:
`GET /api/v1/stats/trend` 的时间聚合桶，首版只支持 `hour` 和 `day` 两种粒度，按 `Asia/Shanghai` 时区计算，并补零返回完整时间轴。
_Avoid_: 任意粒度, 服务器本地时区

**日志分页**:
`GET /api/v1/logs` 使用 `offset` 分页，默认 `limit=20`，最大 `limit=100`，默认按 `created_at DESC, id DESC` 排序。
_Avoid_: 游标分页, 未限定上限的全量查询
首版返回完整 token 字段、完整费用字段以及 `pricing_source`，用于排障和计费核对。

**总览排行**:
`GET /api/v1/stats/overview` 首版固定返回前 5 的模型和客户端排行；模型排行按 `tokens DESC, model ASC`，客户端排行按 `total_cost_usd DESC, client_id ASC`。
_Avoid_: 无上限榜单, 不稳定排序

**静态令牌鉴权**:
首版服务端除 `GET /healthz` 外，所有 API 都要求 Bearer token。
_Avoid_: 匿名查询, 混合鉴权策略
静态 token 为必填配置；未配置时服务端启动失败，不允许自动关闭鉴权。

**服务端配置来源**:
首版 server 允许偏离 `server/CLAUDE.md` 的 `viper` 偏好，采用 Go 标准库加环境变量读取配置；当前只覆盖少量运行参数。
_Avoid_: 为少量配置引入复杂配置框架, 未经记录地偏离既有约定
默认 SQLite 路径为 `./server/data/server.db`，并允许通过环境变量覆盖。

**自动建表**:
首版服务端在启动时自动创建或迁移数据库结构，并幂等初始化模型定价数据，而不是依赖外部 migration 工具。
_Avoid_: 手工建表, 独立迁移工具

**定价来源**:
服务端为每条**使用记录**保存命中的定价来源，用于区分精确匹配、最长前缀匹配或全局默认定价。
_Avoid_: 隐式计价, 不可追溯匹配结果
首版枚举值固定为：`exact`、`prefix`、`default`

**最小字段集**:
首版 `usage_reports` 只保留上报、计费、查询所需的最小字段，不提前引入代理请求场景的延迟、状态码或流式指标。
_Avoid_: 过度兼容旧表结构, 提前扩展未来字段

**来源白名单**:
首版服务端采用半严格来源校验：`app_type` 只允许 `claude`，`data_source` 只允许 `session_log`。
_Avoid_: 任意来源值, 自由扩展字符串

**业务时间**:
`created_at` 是客户端上报的业务发生时间，服务端保留该值并做宽松范围校验；`inserted_at` 是服务端写入数据库的到达时间。
_Avoid_: 完全忽略客户端时间, 只靠到达时间统计

## Relationships

- 一份**会话日志**包含零到多条**Token 使用记录**
- 一份**同步状态**只属于一份**会话日志**
- 一条**已上报记录**对应一条**Token 使用记录**
- 一份客户端上报请求只属于一个**客户端标识**
- 一份**客户端标识**和本地 SQLite 数据库都存放在同一个**应用数据目录**
- 一条**Token 使用记录**在服务端会落成一条**使用记录**
- 一份**上报批次**只属于一个**客户端标识**
- 一条**使用记录**的费用由对应的**模型定价**计算得出
- 未命中具体**模型定价**时，一条**使用记录**可以回退到**默认定价**
- 一条**使用记录**存放在**使用记录库**中
- 一份**上报批次**中的 `client_id` 在服务端按**服务端客户端标识**语义处理
- 一次 `POST /api/v1/sync` 的成功结果采用**同步兼容响应**
- 趋势查询的聚合结果由一组**趋势桶**组成
- 日志查询结果遵循**日志分页**约束
- 总览查询包含固定规则的**总览排行**
- 一条**使用记录**会记录其**定价来源**
- **使用记录库**遵循**最小字段集**原则
- 一条**使用记录**的来源必须满足**来源白名单**
- 一条**使用记录**同时具有**业务时间**和服务端到达时间

## Example dialogue

> **开发者:** "如果一份**会话日志**只新增了 10 行，是不是只需要从这份日志对应的**同步状态**继续读取？"
> **领域专家:** "是，并且只有满足条件的 assistant 消息才会形成新的**Token 使用记录**；上报成功后再把它标记为**已上报记录**，请求里还要带上稳定的**客户端标识**，这些元数据都放在独立的**应用数据目录**里。服务端收到后，会把它们作为**上报批次**落进 `usage_reports` 这张**使用记录库**，并按**模型定价**计算费用。"

## Flagged ambiguities

- "记录" 可能同时指代日志行、提取结果和上报对象；本项目统一用**Token 使用记录**表示可上报的数据单元。
- 服务端领域中统一使用**使用记录**表示已入库数据，不再沿用 `proxy_request_logs` 这类代理语义命名。
- `client_id` 在服务端表示稳定技术标识，而不是机器名或展示别名。
- `POST /api/v1/sync` 是协议兼容特例，其他 API 仍使用通用 `{ "data": ... }` 成功响应。
