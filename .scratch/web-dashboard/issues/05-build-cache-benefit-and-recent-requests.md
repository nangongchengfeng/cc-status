# 落地缓存效益分析与最近请求表格

## What to build

在大屏页面中实现缓存效益分析模块和最近请求表格。该切片需要消费 dashboard 接口中的缓存节省与成本数据，并同时复用现有 `/api/v1/logs` 查询展示最近若干条**使用记录**。目标是让页面既能展示缓存的亮点价值，又能保留最近请求的细节视图，形成完整的大屏闭环。

## Acceptance criteria

- [x] 页面展示缓存效益分析模块，至少能体现节省金额、缓存读取成本和缓存建设成本。
- [x] 页面展示最近请求表格，并复用现有日志接口的排序结果展示最新记录。
- [x] 最近请求表格至少包含时间、模型、输入/输出 token、总费用、客户端标识等核心列。
- [x] 缓存效益分析和最近请求列表都随时间范围切换而刷新到正确数据。
- [x] 当日志列表或缓存数据为空时，模块展示稳定空态而不是异常布局。
