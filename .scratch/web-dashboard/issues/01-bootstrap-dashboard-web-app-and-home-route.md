# 初始化大屏前端脚手架与首页路由

## What to build

在 `web/` 中落地首版数据大屏前端脚手架，并让首页路由直接进入仪表盘页面。该切片需要完成前端运行骨架、基础页面结构和深色大屏主题基线，但不要求一次性完成所有图表与数据接入。目标是让项目具备可运行的前端入口，并明确该应用是一个面向展示的单页数据大屏，而不是通用后台管理台。

## Acceptance criteria

- [x] `web/` 可作为独立前端应用启动和构建，并遵循当前 `web/CLAUDE.md` 约定的技术栈与目录结构。
- [x] 首页路由 `/` 直接进入仪表盘主页面，不额外引入 `/dashboard` 作为首版主入口。
- [x] 页面建立深色大屏的基础布局和主题基线，至少包含页头、主体内容区和卡片式容器骨架。
- [x] 前端基础工程已准备好后续接入 API、图表和表格所需的目录与公共基础设施。
- [x] 首版仍保持单页目标，不扩展出多页面后台管理逻辑。
